# Workflows

Operational playbooks. Each one is a step-by-step walk-through of a
task an argos operator actually does, linked to the feature pages
for background.

If you are new, start with [Add a host](add-host.md) — the rest of
the workflows assume you have at least one working host in the
panel.

| Task | When you reach for it |
|---|---|
| [Add a host](add-host.md) | Bring a new service behind argos end-to-end. |
| [Publish with SSO](publish-with-sso.md) | Put an existing host behind OIDC + ForwardAuth so users authenticate once at the panel. |
| [Respond to an attack](respond-to-attack.md) | Someone is hammering your host. What to look at, what to ban, what to tune. |
| [Onboard an admin](onboard-admin.md) | Add a second human that can operate the panel (OIDC-provisioned). |
| [Restore from backup](restore-backup.md) | Bring a fresh install to the last known-good state. |
